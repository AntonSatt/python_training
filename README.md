## Suggested Approach

1. **Read the README** for the project fully before writing any code.
2. **Start with the core requirements** — get it working before touching stretch goals.
3. **Run often** — test small pieces as you go, don't write 100 lines before running.
4. **Google freely** — looking things up is part of coding, not cheating.
5. **Do the stretch goals** if you're having fun — that's where the real learning happens.

Good luck and have fun!
